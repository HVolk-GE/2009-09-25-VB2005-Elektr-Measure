Aufgabenbereich:

 

Die hier beschriebene Anwendung soll;

Messdaten, die über eine Bügelmessschraube und/oder Waage erfasst wurden in eine Vorlage – Datei schreiben und abspeichern, diese Vorlage – Datei dient zu gleich als Vorlage für Messpunkte auf Bremsscheibe bzw. Bremsbelag und als Vorlage der Messzeitpunkte, der Messungen in dem entsprechenden Prüfprogramm. In dieser Vorlage – Datei kommen noch weitere manuell einzutragenden Werte zum tragen, diese sind jedoch vom Anwender „manuell“ in die entsprechenden Felder einzugeben.

Bei einer fehlerhaften Eingabe – Eingabe fehlt – kann der Anwender die Anwendung nicht erfolgreich beenden.

 

Vorlagen:

Es bestehen Messvorlage – Dateien im Microsoft Excel Format für CV und LV.

 

Der Unterschied:

 

Die LV Microsoft Excel Dateien bestehen zu einem großen Teil aus Microsoft Excel Vers. 4.0 Dateien, auf Grund Ihrer Version, bestehen diese aus nur einem Tabellenblatt.

Wobei dieses Verfahren, „das alte Verfahren“ darstellt, es muss jedoch beachtet werden,

da z.Zt. mit „Microface – Dynograph“, das „nur“ Microsoft Excel Dateien der Vers. 4.0 benutzen kann, im LV Bereich ausgewertet wird.

Dieses stellt heute noch die Standard Auswertung im LV Bereich dar.

Jedoch gibt es auf LV und auf CV Seite, Microsoft Excel Dateien > Vers. 4.0, diese sollen in der Zukunft, im LV Bereich, als Standard benutzt werden.

Im CV Bereich werden diese schon heute als Standard benutzt.

Die letzt genannten Microsoft  Excel Dateien (> Vers. 4.0), bestehen aus mehreren Tabellenblättern, hier wird je ein Tabellenblatt für Bremsbelag innen, Bremsbelag außen und Bremsscheibe angelegt.

Das Format dieser Microsoft Excel Tabellenblätter wird unter Anhang I beschrieben.

 

Schritt für Schritt (Die Benutzung)

 

 

1. Dialog )

Der Anwender soll seine Initialen eingeben – müssen,

Danach kann er Auswählen,

  • Neuer Test – Vorlage auswählen und Testnummer vergeben *,
  • Vorhandener Test - Testnummer auswählen,

 

2. Dialog )

Messzeitpunkt und Messobjekt auswählen,

Der Anwender kann aus den noch nicht gemessenen Messzeitpunkte auswählen, ist ein Messzeitpunkt für einen Bremsbelag gewählt worden wird dieser automatisch für den zweiten Bremsbelag mit übernommen. Übersprungene Messzeitpunkte werden nicht mehr zur Auswahl angeboten.

 

3. Dialog)

Abhängig der Auswahl, die im Dialog 2 getroffen wurde (Bremsscheibe bzw.

Bremsbelag), wird der Dialog 3 angezeigt + die Messpunkte hierfür, die

gemessen werden müssen. Vor der Messung muss die Laufrichtung ausgewählt werden,

wenn alle Messpunkte gemessen wurden, wird dem Anwender eine Meldung angezeigt.

Als nächstes Bremsbelag/Bremsscheibe wiegen.

Wenn alles ausgefühlt wurde, kann der Dialog geschlossen werden.

Beim schließen des Dialoges werden die Daten an Microsoft Excel gesendet und nach dem schließen des Dialoges 2, in Microsoft Excel (der Version der geöffneten Datei) gespeichert.

 

 

 

Der Dialog 2 kann jederzeit geschlossen werden.

 

*) Wenn im dieses konfiguriert wurde.

Die Vorgehensweise im einzelnen;

Der Anwender muß bevor er den Bremsbelag vermessen kann, die Laufrichtung „<------„ oder

„------>„ auswählen. Dann kann der Anwender mit einem Mausklick auf die Schaltfläche „Start Messung“ die Messung des Bremsbelages initialisieren. Durch den Klick auf den besagten Button, wird der aktuell zu messende Messpunkt „gelb“ hervorgehoben.

Nach empfang des Meßwertes , wird die („gelbe Markierung“) Einfügemarke auf die nächst zu vermessende Position gebracht. Dieses wiederholt sich solange es Messpunkte, die noch nicht gemessen wurden, vorhanden sind.

Am Ende wird dem Anwender ein entsprechender Dialog angezeigt, somit ist die „Vermessung“ des aktuellen Bremsbelages beendet.

 

Nun müssen noch die Einträge für;

Messinstr. ID (Bügelmessschraube ID),

Waage ID,

Gewicht, mit einem Mausklick auf die Schaltfläche „Start Wiegen“ wird auf einen Meßwert der angeschlossenen Waage gewartet.

 

Die Reihenfolge, messen <--> wiegen, ist nicht fest vorgegeben, der Anwender kann frei entscheiden in welcher Reihenfolge er diesen Dialog abarbeiten möchte.

 Anwendung / Konfiguration:

 

Um die Anwendung so flexible wie möglich zu halten, musste es möglich sein, verschiedenste Konfigurationen, vom Anwender durch zuführen.

 

Zu konfigurierende Kategorien:

 

  1. Messmittel Konfiguration
  2. Pfad angaben
  3. Standard Konfiguration
  4. Microface Konfiguration

 

Im Hauptmenu über den Menupunkt „Konfiguration/Anwendung konfigurieren“

erreicht man den Konfiguration Dialog.

Dieser besteht aus den oben angegebenen Kategorien, für jede Kategorie ein Blatt.

 

1.) Messmittel Konfiguration :

 

(Messmittel Konfiguration)

 

Links werden die einzustellenden Werte dargestellt, editierbar sind diese nur über das anklicken des Optionsfeldes „Edit“ dieses macht die Einstellungsfelder zugänglich (Editierbar).

 

Links von Oben nach Unten:

„Aktuelle Konfiguration“, stellt die zur Zeit aktuelle Konfiguration dar, hier kann man sehen welcher Anschluss mit welchem Geräte verbunden ist.

„Gefundene Port(s)“, zeigt alle an dem aktuellen Computer verfügbaren Anschlüsse an.

Um einem Messmittel einen neuen Anschluss zu zuweisen, muss man zuerst im Feld

„Aktuelle Konfiguration“ ein Messmittel auswählen, dann über den „Edit“ – Optionsfeld das Feld „Gefundene Port(s)“ editierbar machen und dort den Port selektieren an dem das Messmittel angeschlossen ist, mit dem Button „Speichern“, der nach dem editierbar machen der Felder aktiv ist, bestätigt man seine Änderung und diese werden gespeichert.

 

Die Felder „Instrumenten ID Anschluss....“:

Um diese zu editieren, muss das unterer „Edit“ Optionsfeld ausgewählt werden.

Wenn dieses aktiv ist, kann man in die drei Feldern die aktuellen Messinstrumenten ID Eintragen / Ändern, auch hier bestätigt man seine Eingabe mit dem Button „Speichern“.

 

2.) Pfad Angabe :

 

(Pfad Angabe)

 

Von Links oben nach unten:

„Bearbeitbare Dateien Pfad“, stellt den Pfad, zu den vom Anwender im Dialog „Anmelden/Test Auswahl“ an zu zeigenden Dateien, dar.

Hier befinden sich die zur Zeit, in Arbeit befindlichen (aktuellen), Test / Verschleißwerte.

 

„Template Dateien Pfad“, in diesem Pfad befinden sich die Vorlagen, die der Anwender im Dialog „Anmelden/Neu Anlegen/Vorlage Auswahl“ auswählen kann.

 

Also die Vorlagen die zum Messen/wiegen abgelegt wurden und mit deren Hilfe man einen

neuen Test erstellen kann*.

 

Datei Typ CV/LV), hier kann man festlegen welchen Dateientyp der Anwender in den Auswahllisten angezeigt gekommen soll (zur Zeit ist nur das Microsoft Excel Format verfügbar (*.xl* ; *.xlt ; *.xls))

 

Auch in diesem Dialog sind Änderungen nur über das Optionsfeld „Edit“ möglich und die Bestätigung erfolgt über den Button „Speichern“

 

3.) Standard Konfiguration

 

(Standard Konfiguration)

 

Von Links oben nach unten, dann rechts.

Excel sichtbar, wenn aktiv dann wird Microsoft Excel während der Laufzeit angezeigt.

(*) Kopieren sichtbar, wenn aktiv, kann der Anwender neue Tests aus Vorlagen erstellen, ansonsten kann der Anwender nur bereitgestellt Tests bearbeiten (Dialog Anmelden).

Diese Werte, werden in Realzeit geändert und gespeichert.

 

Die folgenden Werte können nur über das Optionsfeld „Edit“ und als Bestätigung dem Buttton „Speichern“ verändert werden:

 

Anfangszeichen CV Testnummer, der aktuelle für CV Testnummern zu beginn stehende Buchstabe.

 

Anfangszeichen LV Testnummer, der aktuelle für LV Testnummern zu beginn stehende Buchstabe.

CV Tabellennamen, hier werden die einzelnen Tabellennamen (bei aktuellen Excel Versionen > 4.0 :1x Bremsscheibe, 1x Bremsbelag innen und 1x Bremsbelag außen)  angegeben.

 

LV Tabellennamen siehe „CV Tabellennamen“

(Microface Konfiguration)

Dieser ist wie folgt festgelegt;

Jeder Test der mit Dynograph ausgewertet wird, hat eine Excel Tabelle der Version 4.0,

in dieser Tabelle wurde beginnend mit der Zelle „Z2“ eine Tabelle angelegt, diese Tabelle:

 

(LV Microface Tabelle)

 

Wie man sieht befinden sich alle Informationen in einer einzelnen Tabelle, um die Koordinaten zu den Werten zu ermitteln benötigt man hier einige Informationen. Diese Informationen werden in diesen hier aufgeführten verfahren abgelegt und gespeichert.

 

Links oben, Auswahlliste „Template Name“, hier kann man die schon angelegte Microface Templates und deren Einstellungen auswählen und ansehen..

 

Auf der rechten Seite, kann man nun neue Microface Templates anlegen. In der Mitte befindet sich ein Optionsfeld, dieses kann – wenn auf der linken Seite ein Template ausgewählt wurde,

die Einstellung von links in das neu zu erstellende Template übernehmen.

Um Verwechslungen aus zu schließen, hier werden keine „Dateien“ erstellt, nur die Einstellung für die „Template“ Dateien von (für) Microface !

„Beschriftung Pad inside“, welcher Text steht in der Zeile, unter der die Werte für die Vermessung des Inneren Bremsbelages stehen. 

 

„Beschriftung Pad outside“, welcher Text steht in der Zeile, unter der die Werte für die Vermessung des Äusseren Bremsbelages stehen. 

 

„Beschriftung Disc“, welcher Text steht in der Zeile, unter der die Werte für die Vermessung der Bremsscheibe stehen. 

 

„Anzahl der Messzeitpunkt Pad“, wie oft wird der Bremsbelag vermessen.

 

„Anzahl der Messzeitpunkt Disc“, wie oft wird die Bremsscheibe vermessen.

Bremsbelag und Bremsscheibe, die zu vermessenden Punkte und auszufüllenden Felder, sind farblich gekennzeichnet, so das der Anwender direkt sieht welchen Punkt er noch nicht abgearbeitet hat.

Die farblich Kennzeichnung, des gerade aktuellen Messpunktes, wurde umgestellt auf Gelb.

Der Benutzer kann nun, aus einer Liste von Messinstrumenten ID’s auswählen, über eine Combobox.

Bremsscheiben ID, ist auch nachträglich editierbar (im Dialog).

Es ist nicht möglich, eine fertig gestellte Messung zu überschreiben, jeder Messzeitpunkt kann nur einmal gemessen werden.

Nach z.B. Bremsbelag innen wird automatisch der gewählte Messzeitpunkt für Bremsbelag außen angewählt, so das der Anwender gezwungen ist, auf den zweiten (zu vermessenden) Bremsbelag den selben Messzeitpunkt zu vermessen.
